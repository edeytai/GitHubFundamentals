GitHub

En esta pagina de GitHub se dará un repaso a todos los comandos, y conceptos basicos que se vieron en clase sobre GitHub y el flujo de trabajo

¿Qué es Git?

Git es un sistema de control de versiones que permite rastrear cambios en el código, colaborar con otros y recuperar versiones anteriores de un proyecto.

¿Qué es GitHub?

GitHub es una plataforma en la nube que permite almacenar, compartir y colaborar en proyectos que usan Git.

Flujo de trabajo

Primero tendremos que instalar Git en nuestra computadora, los metodos varian dependiendo de que tipo de sistema operativo utilicemos. Tambien tendremos que configurar nuestra identidad y agregar una clave SSH

Para comenzar a utilizar GitHub tenemos que crear una cuenta en GitHub, puedes crear tu cuenta Aqui. Debes crear un repositorio asignándole un nombre y eligiendo si será público o privado. Puedes modificar o agregar archivos directamente desde la plataforma. Cada cambio se guarda con un commit, que registra la historia del proyecto y facilita la colaboración.

Para trabajar con repositorios remotos en GitHub, es esencial clonar el repositorio a tu máquina local. Esto crea una copia completa del proyecto en tu computadora, permitiéndote trabajar de manera independiente. Para clonar un repositorio, utiliza el comando git clone seguido de la URL del repositorio remoto. Una vez clonado, puedes realizar cambios localmente y, posteriormente, sincronizarlos con el repositorio remoto mediante comandos como git push y git pull. Este flujo de trabajo facilita la colaboración y asegura que todos los colaboradores estén actualizados con las últimas modificaciones del proyecto.

A continuación se muestra una tabla para saber que hace cada comando de git que sirve para clonar repositorios, realizar commits, restaurar cambios o cualquier comando importante que te pueda ayudar:

Comando Descripción
git clone [URL] Copia un repositorio remoto a tu máquina.
git add [archivo] Añade el archivo especificado al área de preparación (staging area).
git add -A Añade todos los archivos (nuevos, modificados y eliminados) al área de preparación.
git status Muestra el estado de los archivos (modificados, añadidos o sin seguimiento).
git log Muestra el historial de commits.
git commit -m "[mensaje]" Guarda los cambios con un mensaje.
git push -u origin main Sube cambios a la rama main en el remoto.
git restore [archivo] Revierte cambios no confirmados.
git restore --staged [archivo] Saca el archivo del staging, pero mantiene los cambios.

Es recomendable usar el comando git status despues de cada uso del resto de los comandos para asegurarse de que todo esté en orden.

Tambien podemos trabajar con branches, que nos permiten trabajar en paralelo sin interferir en el trabajo de otros. Una rama es una versión paralela del repositorio principal (main), donde se pueden realizar cambios de forma aislada.

Para esto existen los Pull Requests, que permitien que varias personas trabajen en un mismo proyecto sin conflictos. Los colaboradores crean una rama, hacen cambios y la envían al repositorio remoto para que el líder la revise y fusione en la rama principal.

A continuación se muestra una tabla con los comandos relevantes para trabajar con branches y pull requests:

Comando Descripción
git branch [nombre_rama] Crea una nueva rama llamada [nombre_rama].
git checkout [nombre_rama] Cambia a la rama especificada.
git switch [nombre_rama] Cambia a la rama especificada.
git checkout -b [nombre_rama] Crea y cambia a una nueva rama en un solo paso.
git fetch -a Descarga actualizaciones de todas las ramas sin fusionarlas.
git merge [nombre_rama] Fusiona la rama [nombre_rama] con la rama actual.
git branch -d [nombre_rama] Elimina la rama especificada si ya ha sido fusionada.
git branch -D [nombre_rama] Fuerza la eliminación de la rama, incluso si no ha sido fusionada.
git push -u origin [nombre_rama] Envía los cambios locales de la rama especificada al repositorio remoto

Tambien para los pull requests podemos ocupar GitHub Flow que facilita el desarrollo colaborativo de estos pull requests, esto se hace a traves del repositorio remoto donde estan los pull requests que hacen los demás colaboradores. GitHub Flow te permite revisar y aprovar estos cambios que hacen los demas colaboradores.
